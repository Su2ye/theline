# TheLine（这条线）

> 两个人之间有一条可视的线。线的状态由真实行为驱动——见面频率、距离。零服务器、零成本、纯 P2P。

**[su2ye.site](https://su2ye.site)** — 浏览器打开就能用。添加到主屏幕。

---

## 这是什么？

你和一个人之间，有一条看不见的线。

也许是最好的朋友，也许是恋人，也许是异地很久没见的家人。

**TheLine 把这条线画在屏幕上。** 线的粗细和颜色会随着你们的真实互动而改变——见面越频繁，线越粗越亮；太久没见，它会变细变暗。

没有服务器，没有账号注册，没有数据收集。你们的数据全存在自己手机里，通过浏览器直接点对点通信。

## 核心功能

### 那条线

打开首页就能看到。一条在深色背景上缓缓呼吸的线段，周围漂浮着微光粒子。

- **颜色有温度** — 最近常见面时线是暖橙色，疏远了慢慢变成冷蓝色。不只是粗细变化，颜色也在告诉你们关系的近远
- **两端是你和对方** — 线段两端各有一个光环，里面写着「我」和对方的名字代号
- **状态平滑过渡** — 线从饱满到黯淡不是瞬间跳变，而是慢慢渐变，像真的一样
- **越常见面线越粗** — 过去 30 天里见面次数越多，线条越饱满
- **太久不见线会变** — 超过 14 天没见，线开始抖动变暗；超过一个月，细如发丝；超过两个月，变成灰色

### 见面自动记录

打开页面，GPS 在浏览器前台时工作。

- 当你和对方的距离小于 **50 米** 并持续超过 **15 分钟**，系统自动判断"你们见面了"
- 自动生成一条见面标记，记录时间和地点
- 检测到见面的瞬间，线会**闪亮一下**，手机**震动**，弹出通知「又见面了」

### 提醒对方

点「提醒对方」，对方手机弹出系统通知。点通知直接打开页面。

### 纪念日

配对后点击「在一起」卡片，可以设置你们的纪念日。天数从那天开始算。

### 自动重连

锁屏或切 App 后连接断开，重新打开页面会自动尝试恢复连接，不需要重新配对。

### 见面地图

从首页**向下拉**（像刷朋友圈一样），地图从上方展开。

- 每个见面地点在地图上都有一个标记
- 点击标记可以看详情、写留言、传照片
- 你和对方都能看到彼此写了什么

### 照片解锁

每一条见面标记，你们都可以各自上传一张照片。

- 你上传了，对方看到的是**模糊的**
- 对方也上传了，**双方的照片同时变清晰**
- 这是刻意设计——只有当两个人都愿意分享，记忆才对彼此完整

### 纯粹的点对点

- **不经过任何服务器** — 数据直接在你的手机和对方的手机之间传输（WebRTC）
- **配对只需做一次** — 通过剪贴板交换一段文本，像微信发消息一样简单
- **配对后自动重连** — 下次打开页面，自动尝试重新建立连接
- **所有数据在你手机里** — 标记、照片、留言全存在浏览器本地（IndexedDB）

## 怎么开始？

### 第一步：打开网页

用手机上的 Safari 或 Chrome 打开页面链接，点击「添加到主屏幕」，它就像一个 App。

### 第二步：配对

1. 你点 **「创建配对」**，系统自动复制一段文本到剪贴板
2. 把这段文本发给你要连接的人（微信、短信、AirDrop……随便）
3. 对方打开页面，点 **「加入配对」**，粘贴你发的文本
4. 系统自动生成一段回复文本，对方复制后发回给你
5. 你粘贴回来——**连接建立**

此后不会再需要手动配对，页面会自动重连。

### 第三步：在一起时打开它

TheLine 在**需要见面检测时**打开页面即可。不需要全天候运行——浏览器在后台时无法获取 GPS。

- 约好见面时，双方打开页面
- GPS 检测到距离 < 50 米并持续 15 分钟以上 → 自动记录
- 见面结束后可以关掉，标记已经保存

日常使用时，随时打开页面就能看到那条线和统计数据。

## 隐私说明

| 问题 | 答案 |
|---|---|
| 数据存在哪里？ | 全在你自己的浏览器里（IndexedDB），不经过任何服务器 |
| 服务器能看到我的位置吗？ | 没有服务器。GPS 数据直接在你和对方之间传输 |
| 对方能看到我的实时位置吗？ | 不会。只有当距离足够近时才会自动记录见面，不显示实时坐标 |
| 照片会上传到哪？ | 存在本地浏览器，通过 P2P 传给对方，不经过任何云端 |
| 能被第三方看到吗？ | 信令交换通过加密的 P2P 连接，内容无法被中间人读取 |
| 可以断开配对吗？ | 随时可以。删除浏览器数据即清除所有信息 |

## 常见问题

**「那我们异地怎么办？」**
线会变细、变暗，但它一直在那里。异地时线是你们关系的视觉提醒——等见面那天，它会恢复如初。

**「为什么不用服务器？」**
因为你们的关系不需要第三方过问。技术上讲，纯 P2P 意味着零运营成本，项目可以永久免费运行。

**「需要两个人都打开页面吗？」**
同步数据时需要。数据不经过服务器，只有当两个人同时在线时才能传输。但每个人的本地数据是持久保存的——对方不在线时，你依然能看到历史记录。

**「浏览器切到后台会怎样？」**
这是 PWA 和原生 App 最大的区别：**浏览器在后台时 GPS 和 WebRTC 都会被系统暂停**。见面检测需要在页面前台运行时才工作——见面时打开页面即可，不需要一直开着。

**「通知是实时的吗？」**
是。Web Push 走的是 FCM（Android）和 APNs（iOS 添加到主屏幕后），和微信通知一样，秒到。

**「推送服务器在哪里？」**
在 Cloudflare 全球网络上，免费 24 小时运行。只存储浏览器的推送凭证，不存你和对方的任何内容。

**「URL 被别人知道怎么办？」**
网站是公开的，但数据不是。任何人打开看到的都是空白的线。没有配对码不可能看到你的标记、照片、留言。你的数据存在手机浏览器里，不经过任何服务器。

**「我们俩的数据对方能看到什么？」**
对方能看到你是否在线、你们的见面标记（配对后）以及你在标记里写的留言和上传的照片。你的实时 GPS 坐标不会持续暴露——只有当双方距离小于 50 米且持续 15 分钟以上时才自动生成标记。

**「换手机了怎么办？」**
数据存在浏览器本地，目前没有自动迁移机制。需要在新手机上重新配对，历史标记不会自动同步过来。

**「支持几对人？」**
一对一。这是为两个人之间那条独一无二的线设计的。

**「需要注册吗？」**
不需要。没有账号、没有密码、没有手机号。打开浏览器就能用。

**「可以断开配对吗？」**
随时可以。删除浏览器站点数据即清除所有配对信息和标记。对方那边仍然保留自己的数据。

**「能用多久？」**
只要手机上有浏览器，就一直能用。没有服务器会下线，没有公司会停服。

## 技术实现

> 如果你是开发者，看这里：[DEV.md](./DEV.md)

- **前端框架** React 19 + TypeScript + Vite
- **线段动画** Canvas 2D + requestAnimationFrame，80 段折线 + 2D 噪声驱动有机摆动
- **P2P 通信** 浏览器原生 WebRTC Data Channel，剪贴板方案完成信令握手
- **数据存储** Dexie.js / IndexedDB
- **地图** Leaflet + OpenStreetMap（完全免费，无需 API Key）
- **PWA** 支持添加到主屏幕、离线可用
- **UI** Tailwind CSS + Framer Motion
